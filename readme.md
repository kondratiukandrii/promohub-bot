# 🔥 PromoHub Bot

Telegram-бот для автоматичного продажу промокодів та підписок з **повною автоматизацією**: оплата, видача товару, адміністрування — все без ручної роботи.

## Можливості

### Для покупця
- 🎁 Купівля паків промокодів через кілька крипто-методів
- 🔷 USDT TRC20, 💎 TON, 🟨 USDT BEP20, 🟡 Binance Pay — автоматична перевірка кожні 30 сек
- 📄 Автогенерація PDF з кодами після підтвердження оплати
- 🧾 Команда `/myorders` — історія власних покупок
- 🔗 Реферальна система

### Для власника (адмін-панель прямо в боті)
- 📊 **Статистика** — дохід і продажі за весь час / 24 год / 7 днів, розбивка по методах оплати
- 📦 **Керування паками** — додавання нових паків, нових кодів, видалення паків — все через кнопки, без редагування коду
- 📢 **Розсилка** — надсилання повідомлення всім користувачам бота одним кліком
- 🔔 Миттєве сповіщення адміну про кожну нову покупку

### Технічна надійність
- 🛡️ Anti-fraud захист — кожна транзакція використовується лише один раз
- ⏳ Rate limiting — захист від спаму командами
- 💾 Автоматичний backup бази даних щодня (зберігаються останні 7)
- ⏰ Платежі, що "висять" понад годину — автоматично скасовуються

## Встановлення

### 1. Залежності

```bash
pip install -r requirements.txt
```

Потрібен Python 3.10+

### 2. Налаштування

```bash
cp .env.example .env
```

Заповни `.env`:

| Поле | Обов'язкове | Опис |
|---|---|---|
| `BOT_TOKEN` | ✅ | Токен від [@BotFather](https://t.me/BotFather) |
| `ADMIN_ID` | ✅ | Твій Telegram ID ([@userinfobot](https://t.me/userinfobot)) |
| `CRYPTO_ADDRESS_TRC20` | потрібна хоча б одна адреса | Адреса USDT TRC20 |
| `CRYPTO_ADDRESS_TON` | | Адреса TON-кошелька |
| `CRYPTO_ADDRESS_BEP20` | | Адреса USDT BEP20 (BSC) |
| `TRONGRID_API_KEY` | ⬜ | Безкоштовно на [trongrid.io](https://www.trongrid.io) |
| `BSCSCAN_API_KEY` | ⬜ | Безкоштовно на [bscscan.com](https://bscscan.com/myapikey) |
| `BINANCE_API_KEY` / `BINANCE_SECRET` | ⬜ | Лише для Binance Business акаунту |

Методи оплати, для яких не задана адреса — автоматично не показуються в боті.

### 3. Запуск

```bash
python bot.py
```

Для продакшну — через `systemd`, `tmux`, `screen` або Docker.

## Перший запуск і паки

При першому старті бот сам створює 4 дефолтні паки в базі даних. Далі ВСЕ керування — через адмін-панель у боті:

`/start` → ⚙️ Адмін-панель → 📦 Керування паками

Звідти можна:
- Додати новий пак (ID, назва, ціна, коди)
- Додати коди до існуючого паку
- Видалити пак

Жодного редагування `bot.py` для зміни асортименту більше не потрібно.

## Як працює автоперевірка оплати

1. Покупець обирає пак і метод оплати → отримує QR і адресу
2. Запис додається в таблицю `pending_payments`
3. Фоновий процес кожні 30 сек опитує відповідний публічний API:
   - **TRC20** → TronGrid
   - **TON** → TonCenter
   - **BEP20** → BscScan
   - **Binance Pay** → Binance API
4. Знайдено відповідну за сумою і часом транзакцію → автоматична видача PDF
5. Адміну приходить сповіщення про кожну покупку

## Структура бази даних (SQLite)

- `purchases` — історія покупок
- `income` — дохід з розбивкою по методу
- `tx` — використані хеші транзакцій (anti-fraud)
- `pending_payments` — платежі в очікуванні
- `packs` / `pack_codes` — паки і промокоди (керуються через адмінку)
- `users` — усі, хто хоч раз запустив бота (для розсилки)
- `ref` — реферальні зв'язки

## ⚠️ Важливо перед використанням

- **Перевір живою оплатою** на малу суму перед повноцінним запуском
- Продаж промокодів сторонніх сервісів може порушувати їхні умови використання — врахуй це
- Не публікуй `.env` і не комітьте його в git (вже додано в `.gitignore`)
- Рекомендований хостинг — VPS з постійним з'єднанням (DigitalOcean, Hetzner тощо)
